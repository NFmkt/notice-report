# 공고 리포트 배포 페이지 — Vercel 배포 가이드

## 폴더 구조

```
notice_report_page/
├── index.html          ← 목록 페이지 (현재 placeholder)
├── report.html         ← 리포트 뷰어 (slug 기반 로딩)
├── css/styles.css
├── js/app.js
├── reports/            ← 공고별 JSON 파일 저장 위치
│   └── 20260529-든든전세주택-800호-모집.json
├── to_json.py          ← temp_html.html → JSON 변환 스크립트
├── vercel.json
└── 배포가이드.md
```

---

## 로컬 테스트

브라우저에서 직접 파일 열기는 fetch() CORS 오류가 발생합니다.
간단한 로컬 서버로 실행하세요:

```bash
cd C:\Users\user\my-claude\for_Release\notice_report_page
python -m http.server 8080
```

브라우저에서 아래 URL 접속:
```
http://localhost:8080/report.html?slug=20260529-든든전세주택-800호-모집
```

## 새 공고 리포트 추가

1. notice-auto 실행 완료 후 `temp_html.html` 생성됨
2. 아래 스크립트 실행:

```bash
cd C:\Users\user\my-claude\for_Release\notice_report_page
python to_json.py --title "공고명" --date "YY.MM.DD" --url "https://gobang.kr/notices/번호"
```

3. `reports/YYYYMMDD-공고명.json` 파일 생성 확인
4. Vercel에 폴더 재배포

---

## Vercel 수동 배포 (최초)

1. [vercel.com](https://vercel.com) 로그인
2. "Add New Project" 클릭
3. "Deploy without Git" 선택
4. `notice_report_page/` 폴더를 drag-and-drop
5. 배포 완료 후 생성된 URL 확인

배포 URL 예시:
```
https://your-project.vercel.app/report/20260529-든든전세주택-800호-모집
```

---

## Vercel 재배포 (공고 추가 시)

1. JSON 파일 추가 후 Vercel 대시보드 접속
2. 해당 프로젝트 → "Deployments" → "Redeploy" 또는
3. 폴더를 다시 drag-and-drop
